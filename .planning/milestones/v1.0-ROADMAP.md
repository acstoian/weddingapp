## Milestone Summary

**Key Decisions Made:**

- React-based templates (not plain HTML) — composable, Vercel-native, editable in-browser
- Vercel per-invitation projects via REST API v13 — full automation, instant URLs
- Field-based editor (not Craft.js canvas) — 95% of needs at 10% of cost
- Octokit for git automation — serverless functions can't maintain git working directories
- Branch naming: `invitation/{invitationId}` — avoids multi-invite collisions per user
- DeploymentService abstraction from Day 1 — swap Vercel for alternative without touching call sites
- SSE polling for deployment status — avoids webhook complexity in Phase 1
- Resend for transactional email — official Vercel integration, free tier adequate

**Issues Resolved:**

- Clerk v7 UserButton API change: `afterSignOutUrl` prop removed — dropped prop
- Next.js App Router: root `page.tsx` required but not in plan — added redirect to /dashboard
- Branch naming collision: was `user/{userId}` — changed to `invitation/{invitationId}`
- Vercel project creation: missing `?teamId=` param — fixed
- Re-publish created new URL: `data.url` is per-deployment unique; now reads `data.alias[]` for stable alias
- Post-sign-in redirect: missing `fallbackRedirectUrl` on Clerk SignIn/SignUp — added

**Issues Deferred:**
