    2. `src/lib/services/vercel.service.ts` — implement `VercelDeploymentService`:
       - All methods use `process.env.VERCEL_TOKEN` and `process.env.VERCEL_TEAM_ID`
       - `createOrUpdateProject(invitationId, branch)`:
         - POST to `https://api.vercel.com/v10/projects` with `{ name: "invitation-{invitationId}", gitRepository: { type: "github", repo: "acstoian/weddingapp" } }`
         - If 409 conflict (project exists), GET the existing project instead
         - Return `{ projectId: data.id, url: data.alias?.[0]?.domain || "" }`
       - `triggerDeploy(projectId, branch)`:
         - POST to `https://api.vercel.com/v13/deployments` per RESEARCH.md Pattern 5
         - Use `gitSource: { type: "github", org: "acstoian", repo: "weddingapp", ref: branch }`
         - Return `{ deploymentId: data.id }`
       - `pollStatus(deploymentId)`:
         - GET `https://api.vercel.com/v13/deployments/{deploymentId}`
         - Map `readyState` values: `QUEUED` | `BUILDING` | `INITIALIZING` → `'QUEUED'` or `'BUILDING'`; `READY` → `'READY'`; `ERROR` | `CANCELED` → throw Error
         - Note: This is called in a polling loop by the SSE route (01-07), NOT internally
       - `deleteProject(projectId)`:
         - DELETE `https://api.vercel.com/v9/projects/{projectId}`
